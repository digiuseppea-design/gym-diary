Da cartella locale ad app online

Questo schema mostra il percorso di una app HTML statica: prima viene salvata con Git, poi caricata su GitHub, poi pubblicata online con GitHub Pages usando un workflow.

1

Cartella sul computer

Il progetto vive nella cartella Gym Diary/. Qui modifichi HTML, CSS e JavaScript.

2

Commit con Git

Con git add . e git commit salvi una fotografia della versione attuale.

3

Push su GitHub

Con git push mandi quella versione nella repo online su GitHub.

4

GitHub Actions legge il workflow

GitHub trova .github/workflows/pages.yml e segue le istruzioni scritte li' dentro.

5

Preparazione della cartella _site

Il workflow crea _site e ci copia dentro i file da pubblicare: index.html, app/, mockup/, .nojekyll.

6

Pubblicazione con GitHub Pages

GitHub Pages prende _site e lo rende visitabile dal link pubblico della app.

Cosa ricordare

Git

Tiene traccia delle versioni sul tuo computer.

GitHub

Conserva online una copia della repo.

GitHub Actions

Esegue istruzioni automatiche quando fai push.

GitHub Pages

Pubblica i file come sito web.

Formula mentale 
modifica file
  -> commit
  -> push
  -> workflow
  -> deploy
  -> app online

Il file chiave e' pages.yml: e' la ricetta che dice a GitHub come pubblicare la tua app.
